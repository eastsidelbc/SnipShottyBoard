SnipShottyBoard (SSB) - Development Roadmap 2025
================================================
A lightweight, tabbed sticky note application with clipboard and screenshot support.
Built with WPF .NET 8+ using modern architecture and professional design patterns.

=== PROJECT STATUS OVERVIEW ===
🎯 **Current State**: Phase 4 Complete - Professional Architecture with Drag & Drop
📍 **Next Phase**: Phase 5 - Enhanced UI/UX Features
🏗️ **Architecture**: Clean, modular design with manager classes
💾 **Data System**: Robust JSON persistence with auto-save
🎨 **UI System**: Custom tab management with drag & drop reordering

=== TECHNICAL ROADMAP ===

**Framework & Technology Decisions**
- ✅ **Staying with WPF** - Perfect for desktop-focused sticky notes app
- ✅ **.NET 8+** - Modern runtime with excellent performance
- ✅ **Standard WPF** - No need for enhanced WPF or MAUI for this use case
- 🎯 **Future consideration**: .NET 9 Fluent theming when stable

**📦 STRATEGIC LIBRARY ROADMAP**

**Currently Added (Lightweight Foundation)**
- ✅ **WPF-UI (3.0.5)** - Modern controls and borderless window support
- ✅ **Serilog.Sinks.File (6.0.0)** - Professional logging replacing Debug.WriteLine
- ✅ **CommunityToolkit.Mvvm (8.4.0)** - Better event handling and command patterns

**Phase 1 Upgrades (Add When Ready for Polish)**
- 🎯 **ModernWpf (0.9.6)** - Microsoft's official Windows 11 styling
  - **Why**: Instant professional appearance, matches Windows 11 design system
  - **Impact**: Zero breaking changes, dramatic visual improvement
  - **Add When**: Ready to enhance visual appeal (2-3 weeks)

**Phase 2 Upgrades (Add When Scaling Architecture)**
- 🎯 **Prism.Core (8.1.97)** - Enterprise application architecture framework
  - **Why**: Dependency injection, modular design, MVVM patterns
  - **Impact**: Professional enterprise architecture
  - **Add When**: Adding multiple windows/complex features (1-2 months)

- 🎯 **Microsoft.Extensions.DependencyInjection (8.0.1)** - IoC container
  - **Why**: Professional service management, testable code
  - **Impact**: Foundation for serious enterprise applications
  - **Add When**: Complex business logic and services (2-3 months)

**Future Specialized Libraries (Add as Needed)**
- 🔮 **Hardcodet.NotifyIcon.Wpf** - System tray functionality
- 🔮 **Microsoft.Web.WebView2** - Embedded web browser for help/docs
- 🔮 **SkiaSharp** - Advanced graphics and image manipulation
- 🔮 **Syncfusion Community** - Professional controls (when budget allows)

**Libraries to Avoid (For This Project)**
- ❌ **Material Design WPF** - Conflicts with Windows design language
- ❌ **MahApps.Metro** - Outdated, ModernWpf is better
- ❌ **Enhanced WPF Toolkit** - Somewhat outdated, limited support
- ❌ **DevExpress WPF** - Expensive, overkill for sticky notes app

**Strategic Approach: Gradual Enhancement**
1. **Keep lightweight now** - Focus on core functionality
2. **ModernWpf first** - When ready for professional styling
3. **Architecture libraries** - When complexity demands it
4. **Specialized features** - Only when specific needs arise

**Architecture Principles**
- ✅ **Clean Architecture** - Separation of concerns with manager classes
- ✅ **SOLID Principles** - Single responsibility, dependency injection
- ✅ **Event-driven design** - Loose coupling between components
- ✅ **Testable code** - Structure allows for unit testing

**Performance Targets**
- 🎯 **Startup time**: < 2 seconds cold start
- 🎯 **Tab switching**: < 100ms response time
- 🎯 **Auto-save**: < 500ms save operation
- 🎯 **Memory usage**: < 100MB with 20 tabs and images
- 🎯 **Image handling**: Support up to 50MB total images
