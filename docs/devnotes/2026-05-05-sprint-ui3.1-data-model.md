## Context & Goal

Phase UI-3.1 of Sprint UI-3: add per-image customization properties to
MediaReference and thumbnail size constants to AppConstants. Data-model-only
changes — zero runtime impact.

---

## Changes

### MediaReference.cs — 6 new properties (Schema v3)

| Property        | Type    | Default | Purpose                              |
|-----------------|---------|---------|--------------------------------------|
| Label           | string  | ""      | User-defined label beneath thumbnail |
| ThumbnailSize   | int     | 150     | Container width (Big/Med/Small)      |
| IsHidden        | bool    | false   | Shows placeholder instead of image   |
| ShowLabel       | bool    | false   | Toggle label row visibility          |
| ShowDate        | bool    | true    | Toggle date portion of timestamp     |
| ShowTime        | bool    | true    | Toggle time portion of timestamp     |

`ThumbnailSize` defaults to `AppConstants.ThumbnailSizeDefault` (150px / Big).
Added `using SnipShottyBoard.Data;` to resolve the constant reference.

### AppConstants.cs — 4 new constants

```
ThumbnailSizeBig     = 150   (default)
ThumbnailSizeMedium  = 100
ThumbnailSizeSmall   = 60
ThumbnailSizeDefault = ThumbnailSizeBig
```

Placed in a new `#region Thumbnail Sizes (Schema v3)` nested inside
the existing `#region Image Configuration`.

---

## Why These Defaults

- `ShowLabel = false` — prevents empty label rows on existing thumbnails
- `ShowDate = true, ShowTime = true` — preserves current behavior
- `ThumbnailSize = Big (150)` — matches current `MediaContainerWidth = 150`
- `IsHidden = false` — all existing images remain visible
